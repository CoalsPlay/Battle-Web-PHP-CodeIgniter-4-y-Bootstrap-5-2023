Overview

Battle-Web PHP is a "Social Network" concept, with RPG touches included as main mechanics. It has many functions and features that make it very complete and entertaining, as each user is a "character" in this game, which can progress as if it were a mythical MMORPG but with its web limitation and therefore its non-graphic game style.

Installation

First you need to upload the database that is in the "/DB/" directory to your MySQL server. Once the database is uploaded you will have to configure your data in the Database.php file to make the connection. Once the connection with the database is established, we will go to the .env file to configure the URL of your site in line 23 (example: app.baseURL = 'http://example.com/') and to the App.php file on line 25 (example: public string $baseURL = 'http://example.com/'; (terminated with '/' in both files)). Finally, we'll go to the Email.php file and we'll need to set up your SMTP server credentials. (This is required for the password recovery system to work)

Features

Battle-Web PHP has many systems that give shape to this concept, it has all the basic elements, from its administration that can be managed by a user with permissions, it can execute any function, like adding enemies, objects in the store, categories. And the maps, news, etc. They also have control of the entire application as game parameters and general site information. Later I will detail some systems that are implemented.

- Login and register.
- News and comments system.
- System of enemies with unique attributes and fully customizable.
- System of categories and maps where you can introduce enemies.
- TOP system of the best players of the site.
- 100% Administration where you can manage the application completely.
- Profile system.
- System of combats as much with enemies (PvM) as with other users (PvP).
- Shop and inventory system.
- Private messaging system and notifications.
- Friends system.
- Feedback system to be able to report errors or suggestions directly to the administration.
- Permits with restrictions (for now there are three types of permissions, administrator, mod and taxpayer).
- Customizable and fully dynamic FAQ system from the administration.

Permissions/ranks

Currently Battle-Web PHP has three types of ranks, which would be "Administrator" (1) which has all the possible permissions, we would continue with "Moderator" (2) that would have many areas of the administration restricted and therefore many permissions to general level, and finally we would have the "Collaborator" (3) who would have even more restricted areas, so as the name suggests would only serve as support for the news or help the administration in other ways, their access to the Administration is very limited.

The ranks in BW-PHP are identified by IDs, which i show in parentheses in the previous mentions, and therefore the code understand these ranges only by those IDs.

Enemies, shop and maps

Enemies are part of the mechanics of the application, you can include them in the administration in the section "Enemy Manager", you can also equip each enemy with statistics and unique rewards.

In the store you can also include sales items that can be used as consumables throughout the user's progress. In administration, you can include such items with your name, description, the effect to be granted and the amount of its effect, together with the price.

The maps are the main part of the game, since it is where you will have to face the different enemies that you are implementing, first of all you must create the categories that in a more "gamer" concept would be "Worlds" or "Episodes" , and within those categories you should enter the maps. It should also be noted that in the creation of enemies you must assign a map, so if there are no maps I do not advise creating enemies.

IMPORTANT: the images of the enemies, products of the store and maps, must deposit it in the corresponding directories, which would be "/assets/img/mobs"(for enemies), "/assets/img/items"(for objects of store), "/assets/img/maps"(for maps). Once you have deposited these images, they will be automatically listed in the administration so that you can select your image in a drop-down list.
